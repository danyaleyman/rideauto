# Дизайн-система (код) — RideAuto `web`

Полноценный **дизайн ↔ код контракт** (Figma tokens package, semver DS) здесь не ведётся; ниже — **источник правды в репозитории** и как расширять.

## Где смотреть

| Слой | Файл / каталог |
|------|----------------|
| Семантические цвета, радиусы, шрифты | `web/src/app/globals.css` (`:root`, `.dark`, блок `@theme inline`) |
| «Приподнятая» обводка карточек | CSS-переменная `--elevated-ring`, утилита Tailwind `ring-elevated-ring` |
| Компоненты UI | `web/src/components/ui/*` (shadcn-стиль) |
| Живые примеры | Storybook: `npm run storybook` в `web/` |

## Правила расширения

1. **Новый семантический токен:** добавить `--my-token` в `:root` / `.dark`, при необходимости пробросить в `@theme inline` как `--color-my-token` для `bg-*` / `text-*` / `ring-*`.
2. **Новый компонент:** по возможности собирать из `ui/*`; для нетривиальных блоков — отдельный файл + сторис в `src/stories/`.
3. **Слом обводок/отступов:** избегать новых `ring-black/[0.0x]` в пользу токенов из `globals.css`.

## Версионирование

Версия «DS» = версия приложения / тег релиза. При необходимости отдельного semver для токенов — вынос палитры в npm-пакет `@rideauto/tokens` (не сделано).
