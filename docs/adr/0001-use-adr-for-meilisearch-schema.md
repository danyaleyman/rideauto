# ADR 0001: ADR для изменений схемы Meilisearch

- **Статус:** принято  
- **Дата:** 2026-05-03  

## Контекст

Индекс Meilisearch (`cars`) задаёт фильтры, фасеты, сортировки и дедуп выдачи. Изменения без документа приводят к «тихим» поломкам фронта и расхождению с Postgres.

## Решение

1. Любое изменение `infrastructure/meilisearch/index_settings.json` или полей документа в `sync_meilisearch.py` сопровождается **кратким ADR** в `docs/adr/`, если:
   - меняется поведение поиска для пользователя;
   - добавляется/удаляется filterable/sortable атрибут;
   - меняется `distinctAttribute` или семантика `catalog_dedupe_key`.
2. Владелец схемы: `infrastructure/meilisearch/SCHEMA_OWNERS.md` + CODEOWNERS.

## Последствия

- Дисциплина PR выше; новым участникам проще понять «зачем так».
- Небольшие правки синхронизации без эффекта на контракт можно делать без ADR по усмотрению владельца.
