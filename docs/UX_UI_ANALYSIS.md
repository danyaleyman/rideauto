# Анализ UX и UI — World Ride Auto (Encar)

## Общая оценка

| Критерий | Оценка (1–5) | Комментарий |
|----------|--------------|-------------|
| Визуальная согласованность | 4 | Единые токены в common.css, консистентные карточки и кнопки |
| Удобство навигации | 3.5 | Понятная структура, но нет скелетона загрузки и слабая связь карточка → деталь |
| Фильтрация и поиск | 4 | Мощные фильтры, мультивыбор, каскад; не хватает быстрого сброса и поиска по тексту |
| Мобильная адаптация | 3.5 | Есть брейкпоинты; фильтры на мобиле занимают много места |
| Доступность (a11y) | 3 | Часть aria, фокус есть; нет skip-link, карточка не вся кликабельна |
| Информационная архитектура | 4 | Логичные разделы на карточке авто, хлебные крошки |

**Итог: 3.7/5** — добротная коммерческая вёрстка с чёткой системой дизайна; основной потенциал — в доработке сценариев (загрузка, переход в деталь, мобильные фильтры, доступность).

---

## Сильные стороны

1. **Дизайн-система** — переменные в `common.css` (цвета, отступы, тени, радиусы) дают единый стиль и упрощают правки.
2. **Фильтры** — каскад марка → модель → поколение → комплектация, мультивыбор, счётчики, сортировка по алфавиту; dropdown с панелью вместо «стены» чекбоксов.
3. **Карточки в каталоге** — превью с переключением по hover, цена/налог/доставка, бейдж «Добавлен сегодня», иконки «Поделиться» и «В избранное».
4. **Карточка авто (car.html)** — две колонки (контент + sticky плашка), галерея с миниатюрами, модалки (галерея, расчёт, как оформить), блок «Похожие по цене».
5. **Фокус и интерактив** — `:focus-visible`, тени при фокусе, hover на кнопках и карточках.
6. **Типографика и читаемость** — нормальные размеры шрифта, контраст текста.

## Рекомендации по UX

### Высокий приоритет

1. **Кликабельность карточки авто в каталоге**  
   Сейчас в деталь ведёт только кнопка «Ставка» (tax) и, возможно, часть области. **Сделать всю карточку (или блок с фото + заголовок + цена) ссылкой на `car.html?id=...`**, оставив для кнопок «Поделиться» и «В избранное» `event.stopPropagation()`, чтобы не было двойного перехода.

2. **Состояние загрузки каталога**  
   Пока грузятся `cars.json` и маппинг, сетка пустая. **Добавить скелетоны карточек** (серые плейсхолдеры с анимацией пульса) или спиннер с текстом «Загрузка каталога…», чтобы было понятно, что данные подгружаются.

3. **Пустое состояние фильтров**  
   Сообщение «Ничего не найдено. Попробуйте изменить фильтры» — хорошее начало. **Рядом показывать кнопку «Сбросить фильтры»** (или усиливать визуально уже существующую в панели), чтобы один клик возвращал к полному списку.

4. **Явная обратная связь при смене фильтров**  
   При большом `allCars` применение фильтров может занимать время. **Краткий индикатор** (например, затемнение сетки + «Применяем фильтры…» или счётчик «Найдено N») снижает ощущение «зависания».

### Средний приоритет

5. **Поиск по тексту**  
   Есть только фильтры по полям. **Строка поиска** (марка/модель/VIN) с подсветкой совпадений упростит быстрый поиск конкретного авто.

6. **Сохранение выбора фильтров в URL**  
   При обновлении страницы или передаче ссылки фильтры сбрасываются. **Query-параметры** (`?mark=BMW&model=E-Class`) или `sessionStorage` позволят вернуться к тому же набору фильтров и делиться ссылкой на подборку.

7. **Пагинация + «наверх»**  
   После перехода на следующую страницу каталога **прокручивать в начало списка** (или к заголовку «Автомобили в наличии»). Опционально — кнопка «Наверх» при длинной странице.

8. **Карточка авто: загрузка и ошибка**  
   Вместо одного блока «Загрузка данных…» можно **разбить на скелетоны блоков** (галерея, характеристики, история). При ошибке загрузки — сообщение и кнопка «Повторить» или «В каталог».

### Низкий приоритет

9. **Избранное**  
   Сейчас кнопка «В избранное» без логики. Либо **реализовать** (localStorage + иконка «заполненное сердце»), либо временно убрать или сделать неактивной с подсказкой «Скоро».

10. **Хлебные крошки на car.html**  
    Сделать **второй элемент кликабельным** (название авто → переход на якорь или оставить как есть), проверить обрезку длинных названий и единый стиль.

## Рекомендации по UI

### Высокий приоритет

1. **Мобильное меню фильтров**  
   На ширине &lt; 900px панель фильтров тянется на всю ширину и занимает много места. **Сделать кнопку «Фильтры» (и бейдж с числом активных)**, по нажатию — выезжающая снизу/сбоку панель (drawer) с теми же блоками и кнопками «Применить» / «Сбросить». Так список остаётся главным, а фильтры не «съедают» экран.

2. **Единая точка входа в деталь из карточки**  
   В карточке несколько зон: фото, заголовок, цена, кнопка налога. **Визуально выделить одну основную область** (например, обводка/подсветка при фокусе всей карточки) и гарантировать, что клик по карточке (кроме кнопок) ведёт в деталь — без двусмысленности.

3. **Цена и fallback**  
   Текст «Обратитесь к менеджеру для просчёта» длинный; на мобиле может переноситься. **Сократить до «Цена по запросу»** в интерфейсе, полную форму оставить в тултипе или в детальной карточке.

### Средний приоритет

4. **Баннер (hero)**  
   На узких экранах блок с тремя карточками и слайдером может давить. **Уменьшить высоту/отступы на мобиле**, рассмотреть сворачивание статичных карточек в одну строку с прокруткой или табами.

5. **Пагинация**  
   При большом числе страниц показывать не все номера. **Шаблон «1 … 4 5 6 … 20»** с кнопками «Назад»/«Вперёд» и опционально полем «Страница N из M» улучшит юзабилити.

6. **Иконки в карточке**  
   «Поделиться» и «В избранное» — без текста. Добавить **tooltip при hover** («Поделиться», «В избранное») и при необходимости увеличить зону нажатия до 44px по высоте на тач-устройствах.

7. **Счётчик «Найдено»**  
   Рядом с числом можно выводить **краткую сводку** («Найдено 42 авто по вашим фильтрам») или подсказку при наведении на (i), чтобы было понятно, что учитываются только активные фильтры.

### Низкий приоритет

8. **Микроанимации**  
   Появление карточек при первой загрузке (лёгкий fade-in или slide-up с небольшой задержкой по индексу) и плавное появление панели фильтров на мобиле улучшат восприятие, не перегружая интерфейс.

9. **Тёмная тема**  
   Если в планах — заложить сейчас переменные для `prefers-color-scheme: dark` в `common.css`, чтобы позже подключить тему без переписывания верстки.

## Доступность (a11y)

- **Сделано:** часть кнопок и триггеров с `aria-label` / `aria-expanded`, фокус через `:focus-visible`, семантика заголовков (h1, h2).
- **Добавить:**
  - **Skip-link** в начале страницы: «Перейти к каталогу» / «Перейти к фильтрам», скрытый визуально до фокуса.
  - **Карточка каталога:** обернуть в `<a>` или добавить одну основную кнопку/ссылку с понятным текстом для скринридеров («Перейти к карточке BMW E-Class …»).
  - **Живые регионы:** при смене фильтров обновлять `aria-live="polite"` с текстом «Показано N автомобилей», чтобы скринридер объявлял результат.
  - **Модальные окна:** ловушка фокуса (Tab внутри модалки), закрытие по Escape и по клику на overlay уже обычно есть — проверить и документировать.

## Мобильные устройства

- **Сейчас:** `main-layout` в колонку, фильтры на всю ширину, карточки в колонку, hero перестраивается.
- **Улучшить:**
  - Фильтры в drawer (см. выше).
  - В карточке каталога на мобиле **увеличить зону нажатия** для перехода в деталь (вся карточка или крупная область).
  - В шапке на очень узких экранах **свернуть навигацию в гамбургер**, если пунктов станет больше.
  - Проверить размеры кнопок и полей ввода (минимум ~44px по высоте для основных действий).

## Краткий чек-лист внедрения

| # | Действие | Эффект |
|---|----------|--------|
| 1 | Вся карточка каталога ведёт в car.html (кроме кнопок) | Меньше промахов, привычный сценарий |
| 2 | Скелетон или спиннер при загрузке каталога | Нет ощущения «пустой» страницы |
| 3 | На мобиле — кнопка «Фильтры» + drawer | Больше места списку, удобнее выбор |
| 4 | Skip-link «К каталогу» / «К фильтрам» | Лучше навигация с клавиатуры и скринридером |
| 5 | Короткий текст для «цена по запросу» + тултип | Аккуратный вид на мобиле |
| 6 | После смены страницы пагинации — скролл вверх | Предсказуемое поведение |
| 7 | Сохранение фильтров в URL (опционально) | Можно делиться ссылкой на подборку |

---

Итог: интерфейс уже сильный по структуре и визуалу; фокус стоит сделать на **загрузке, однозначном переходе в деталь, мобильных фильтрах и базовой доступности** — это даст наибольший прирост по UX при разумных трудозатратах.
