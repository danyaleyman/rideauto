# Аудит зрелости: J+K, I, L, M, N

Оценка **по секциям** на момент ревью репозитория: насколько блоки закрыты для «взрослого» продукта (наблюдаемость, единые правила, отказоустойчивость).

## J+K — свежесть, sold/remove, кэш

| Критерий | Статус | Заметки |
|----------|--------|---------|
| `updated_at` в БД и в API | **Сильно** | `_catalog_updated_at` / `catalog_updated_at`; v2 делает timestamp обязательным |
| Единое правило sold | **Хорошо** | Фильтр Meili + гидратация; см. `BLOCK_JK_CACHE_FRESHNESS.md` |
| Кэш без «вечного старья» | **Хорошо** | TTL + эпоха + инвалидация |
| Фронт использует свежесть | **Частично** | Типы и дедуп по `catalog_updated_at`; полноценный клиентский кэш по timestamp — по желанию |

**До полноты взрослого продукта:** мониторинг «доля ответов без `catalog_updated_at`», алерт на рассинхрон индекса и БД по sold.

## I — канон фасетов, анти-мусор, drift

| Критерий | Статус | Заметки |
|----------|--------|---------|
| Единый канон | **Хорошо** | `FACET_SPECS_MEILI`, `facet_normalize`, `facet_canonical_english` |
| Анти-мусор | **Частично** | Числовой мусор в transmission, синонимы KO; нет глобальных порогов частоты по всем осям |
| Drift | **Хорошо** | Расширенные тесты в `test_facet_specs_drift.py` + контракт `FacetsResponse` |

**До полноты:** снапшоты распределений или allowlist префиксов по доменам; чёрные списки опечаток вне кода (конфиг/таблица).

## L — дедуп

| Критерий | Статус | Заметки |
|----------|--------|---------|
| UI | **Есть** | VIN-дедуп на каталоге (`catalog-vin-dedupe.ts`) |
| Индекс Meili | **Улучшено** | `catalog_dedupe_key` + `distinctAttribute`; см. `BLOCK_L_DEDUP.md` |
| Postgres | **Частично** | Нет обязательной канонической одной строки на авто; слияние дублей — по желанию |

**До полноты:** job слияния в Postgres, алерты на рассинхрон; проверка total/пагинации с distinct на проде.

## M — масштаб и стоимость

| Критерий | Статус | Заметки |
|----------|--------|---------|
| Кэш и TTL | **Хорошо** | Задокументировано в `BLOCK_M_SCALE_COST.md` |
| Метрики | **Улучшено** | Histogram размера ответа + latency; шаблоны PromQL в `BLOCK_M_SCALE_COST.md` |
| Бюджеты / алерты | **Частично** | Правила в Alertmanager / Grafana — вне репозитория |
| Профилирование | **Вне репо** | Рекомендуется APM по маршрутам search/car/facets |

**До полноты:** зафиксировать пороги алертов в IaC; опционально tracing.

## N — governance

| Критерий | Статус | Заметки |
|----------|--------|---------|
| Версия API + golden | **Сильно** | v1/v2, тесты контракта |
| Runbook | **Хорошо** | `BLOCK_N_GOVERNANCE.md`, J+K, clean-read |
| Чеклист релиза | **Улучшено** | `backend/docs/RELEASE_CHECKLIST_CATALOG.md` |
| Владелец схемы Meili | **Улучшено** | `infrastructure/meilisearch/SCHEMA_OWNERS.md` (+ рекомендация CODEOWNERS) |
| Аудит схемы индекса | **Процесс** | Drift-тесты фасетов + контракт `index_settings` |

**До полноты:** реальные GitHub teams в CODEOWNERS; ADR для спорных изменений индекса.

## Вывод

Репозиторий **ближе к зрелому продукту** в **контракте API (F/G, v2), кэше (J+K), каноне фасетов (I) и дедупе выдачи в Meilisearch (L)**. Дальше всего до «взрослого» операционного контура — **алерты и SLO в бою (M)**, **слияние дублей в Postgres (L)** и **приёмочные проверки пагинации/total с distinct**. **N** дополнен чеклистом релиза и владением схемой индекса; остаётся назначить реальные владельцев в GitHub.
