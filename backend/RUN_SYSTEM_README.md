# Encar System Runner

Универсальный скрипт-обертка для запуска системы Encar Parser с PostgreSQL.

## 🚀 Возможности

Этот скрипт предоставляет простой интерфейс для всех операций системы:

- **Автоматическая проверка зависимостей** и установка при необходимости
- **Проверка конфигурации** и создание при отсутствии
- **Тестирование системы** перед запуском
- **Единый интерфейс** для всех операций
- **Подробное логирование** всех действий
- **Обработка ошибок** и информирование о проблемах

## 📋 Команды

### Стандартный запуск (рекомендуется)
```bash
python backend/run_system.py
```
Запускает полную проверку системы и ежедневное обновление.

### Конкретные операции

#### Настройка системы
```bash
python backend/run_system.py --setup
```
Полная настройка системы: проверка зависимостей, создание конфигурации, тестирование.

#### Тестирование системы
```bash
python backend/run_system.py --test
```
Только тестирование системы без запуска обновления.

#### Ежедневное обновление
```bash
python backend/run_system.py --daily
```
Запуск ежедневного обновления (только новые/измененные автомобили).

#### Полный прогон
```bash
python backend/run_system.py --full
```
Запуск полного сканирования всех автомобилей.

#### Умное обновление (рекомендуется)
```bash
python backend/run_system.py --smart
```
Автоматическое определение режима обновления на основе состояния базы данных.

**Как работает умное обновление:**
- **Пустая база данных** → Запуск `parser_full.py` (первоначальная загрузка)
- **База есть, но сканирования не было** → Полное обновление
- **Данные устарели >7 дней** → Полное обновление
- **Данные устарели >1 дня** → Ежедневное обновление
- **Данные актуальны** → Ежедневное обновление

#### Настройка автоматического обновления
```bash
python backend/run_system.py --auto
```
Настройка cron (Linux/macOS) или Task Scheduler (Windows).

#### Быстрый запуск
```bash
python backend/run_system.py --quick
```
Только обновление без предварительной проверки (если система уже настроена).

### Дополнительные параметры

#### Количество потоков
```bash
python backend/run_system.py --daily --workers 10
python backend/run_system.py --full --workers 5
```
Настройка количества потоков для парсинга.

## 🔄 Рабочие процессы

### Для первого запуска
```bash
python backend/run_system.py --setup
python backend/run_system.py --test
python backend/run_system.py --full
```

### Для повседневного использования
```bash
# Стандартный режим (рекомендуется)
python backend/run_system.py

# Или быстрый запуск (если система уже настроена)
python backend/run_system.py --quick
```

### Для отладки
```bash
# Проверка системы
python backend/run_system.py --test

# Ежедневное обновление с 2 потоками
python backend/run_system.py --daily --workers 2
```

### Для автоматизации
```bash
# Настройка автоматического обновления
python backend/run_system.py --auto

# Или добавление в cron вручную
echo "0 3 * * * cd /path/to/project && python3 backend/run_system.py --daily --workers 5 >> run_system_cron.log 2>&1" | crontab -
```

## 📁 Файлы логов

Скрипт создает логи в двух местах:

- `run_system.log` - Основные логи скрипта
- `auto_update.log` - Логи обновления системы
- `run_system_cron.log` - Логи cron заданий (если используется)

## ⚠️ Обработка ошибок

Скрипт автоматически обрабатывает следующие ситуации:

1. **Отсутствие зависимостей** - автоматическая установка
2. **Отсутствие конфигурации** - автоматическая настройка
3. **Ошибки подключения** - информирование и прекращение работы
4. **Прерывание пользователем** - корректное завершение
5. **Непредвиденные ошибки** - логирование и информирование

## 💡 Советы

1. **Для первого запуска** всегда используйте `--setup` для полной настройки
2. **Для повседневного использования** используйте стандартный режим `python backend/run_system.py`
3. **Для отладки** используйте `--test` и `--workers 2` для более медленной и детальной работы
4. **Для автоматизации** используйте `--auto` или настройте cron вручную
5. **При проблемах** всегда проверяйте логи: `run_system.log` и `auto_update.log`

## 🚨 Важно

- Скрипт требует Python 3.8+
- Для работы с PostgreSQL нужен установленный psycopg2-binary
- Конфигурационный файл `config.json` создается автоматически при первом запуске
- Все операции логируются для последующего анализа
- При ошибках скрипт показывает подробную информацию для диагностики
