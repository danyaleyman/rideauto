# Резервное копирование SQLite (миграция / старые снимки)

Актуальный продакшен каталога — **PostgreSQL**. Этот документ полезен, если у вас остались файлы `encar_*.db` или вы делаете off-line снимок перед `migrate_sqlite_to_postgres.py`.

## Режим WAL

API и скрейперы обычно включают `PRAGMA journal_mode=WAL`. Рядом с файлом БД появляются:

- `имя.db-wal` — журнал записей;
- `имя.db-shm` — shared memory.

Для **консистентной** копии одного файла `.db` в активной записи:

1. Остановите процессы, которые пишут в БД (API, импорт), **или**
2. Выполните онлайн-бэкап через SQLite (см. ниже).

Копирование только `encar_cars.db` или **`encar_china.db`** (Dongchedi) без `-wal`/`-shm` при активной записи может дать **повреждённый** снимок — для каждого файла каталога выполняйте отдельный `.backup` (см. выше).

## Онлайн-бэкап (рекомендуется)

На сервере, где есть `sqlite3` CLI:

```bash
sqlite3 /path/to/encar_cars.db ".backup '/path/to/backup/encar_cars_$(date +%Y%m%d).db'"
```

Или из Python (кратко): открыть исходную БД и вызвать `backup` API целевого соединения.

## Полное копирование в простое

Если записи нет (сервис остановлен), скопируйте все три файла, если они есть:

- `encar_cars.db` (+ `-wal`, `-shm` при необходимости)
- `encar_china.db` (+ `-wal`, `-shm`) — китайский каталог Dongchedi

## Снапшот тома

На VPS с LVM/снапшотами диска можно делать снимок тома с БД, затем копировать файлы из снапшота — это согласовано на уровне ФС.

## Место на диске

Каталог ~500k строк: два файла БД (`encar_cars.db`, `encar_china.db`) могут занимать **несколько GB** каждый; в WAL-режиме дополнительно растут `*-wal` (до сжатия checkpoint). Заложите запас на том же разделе, что и бэкапы, и мониторьте `df`.

## После крупного импорта

Если обновляли каталог в обход штатного API (сырой `INSERT`/восстановление), имеет смысл один раз открыть БД и выполнить `ANALYZE cars;` — планировщик SQLite лучше оценит селективность (в штатном старте API индексы и `ANALYZE` поднимаются через `_ensure_catalog_indexes`).

## Расписание (если ещё ведёте SQLite)

Настройте `cron` или systemd timer: ежедневный `.backup` в отдельный каталог + ротация (удаление копий старше N дней).

Файловый sitemap из **Postgres**: `deploy/systemd/prod-encar-sitemap-gen.timer` и `scripts/generate_sitemap_files.py` (в окружении — `DATABASE_URL`). Проверяйте свежесть `sitemap-index.xml` и шардов в каталоге `--out`.

## Проверка после восстановления / миграции

1. Импортируйте снимок `.db` в Postgres через `migrate_sqlite_to_postgres.py` (копия на тестовой машине).
2. Проверьте FastAPI: `GET /api/health`, `/api/cars?per_page=2`, `/api/car/{id}`.
3. Зафиксируйте время прохождения и отклонения в runbook.
